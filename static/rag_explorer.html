How It Was Built

1. Data Layer
5,000 synthetic JIRA-format VWO test cases were generated into a CSV corpus.
2. Ingestion Layer
Upload, schema preview, text/meta column selection, chunking, embedding, indexing.
3. Retrieval Layer
Dense and sparse retrieval run in parallel, then fused via Reciprocal Rank Fusion.
4. Precision Layer
Top candidates are reranked with BAAI cross-encoder for better relevance.
5. Generation Layer
LLM answers with chunk-grounded context and explicit citations.

Key Components

  • app.py - Flask API + UI pages + SSE stage updates.
  • ingest.py - Optional CLI uploader and ingest trigger.
  • Qdrant collection - vwo_test_cases with dense+sparse vectors.
  • /upload - Data onboarding and feature selection.
  • /ingest - Live pipeline visibility.
  • /chunks - Corpus browser with metadata filters.
  • /chat - Rewrite, retrieve, rerank, generate flow.

Modes in Chat

Answer mode: grounded Q&A over indexed test cases with [Chunk N] references.

Generate mode: prompts like "create a new test case for JIRA VWO-1234" return a structured testcase template derived from retrieved examples.

Compatibility: Works with existing CSVs once uploaded and ingested, as long as core fields such as title, steps, expected, priority, module, and jira_id are present.

Teaching Demo Script

1) Upload testcase CSV 2) Select text/meta columns 3) Start ingest and explain each stage 4) Open chunks and inspect sparse/dense previews 5) Ask questions in chat, show rewrites + rerank 6) Trigger generate mode and compare output quality
